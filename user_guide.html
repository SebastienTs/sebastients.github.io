LOBSTER

Getting Started

Last updated: November 13th, 2019

LOBSTER is an image analysis environment to identify biological objects in microscopy images and measure their spatial location, geometry, dynamics and intensity distribution. Multiple images can be batch processed in one go and the software sets no limit on image size. The objects can be exported as 2D/3D models importable in external software, for instance for exploration or simulation of the process under study.

Setting Up

Assuming that MATLAB (version >=2015a) and both Image processing and Statistics & Machine Learning toolboxes are installed, setting up LOBSTER is extremely simple:

  • Download the software from its GitHub repository (green button)
  • Unzip content to an empty folder where MATLAB has read/write access
  • Create an empty folder /Images inside installation folder and unzip the content of this archive inside (sample images)
  • Create an empty folder /Results inside installation folder, and unzip the content of this archive inside (sample workflow results)
Windows is highly recommended but LOBSTER should be compatible with all major operating systems. Refer to documentation for details (sections Requirements, Installation and Startup).

Initializing

To initialize LOBSTER:

  • Launch MATLAB
  • >> cd E:/LOBSTER (navigate to local installation folder)
  • >> init (initialize)
This action must be performed before using LOBSTER. The whole procedure is illustrated at the beginning of this video.

Architecture

Objects identification, measurements and modeling are performed sequentially and the results are stored in simple formats after each step. These operations are respectively performed by three computational modules called from MATLAB console (scripting) or LOBSTER Panel (Graphical User Interface). In this tutorial we cover the Panel, refer to this page for scripting.

JENI: Identifying objects

All TIFF images from an input folder are typically processed into object masks by applying a sequence of image analysis functions (workflow) configured in a journal (short text file). The object masks can be of four kinds illustrated in the figure above:

  • Binary masks ('Objs'): objects encoded as connected groups of foreground pixels
  • Skeleton masks ('Skls'): 1-pixel curves marking filament centrelines
  • Seed masks ('Spts'): foreground pixels mark objects
  • Label masks ('Trks'): objects encoded as connected groups of pixels with same intensity level (unique ID).

IRMA: Measuring object masks

Object masks can be used to measure objects or become the basic building blocks of more complex workflows. For instance, in the figure above Object Mask 1 is both used to measure C01 intensity and count the number of C02 objects inside C00 objects. Objects masks can also be used for object based co-localization study, or to extract relative object distance distributions (e.g. spots to filaments). All object measurements are performed by IRMA from the object masks, and results are exported to .csv reports.

JOSE/IRMA: Exporting object models

Identified objects can be exported as 2D/3D models so as to be explored externally: filaments networks are exported as SWC or OBJ models, object surfaces are exported as STL meshes and object tracks are exported as SWC trees. Finally, object markers can be exported as CellInsight files (.xls), an ImageJ macro (provided in /Tools folder) enabling to edit and visualize them.

Identifying Objects

Image analysis workflows to identify objects are configured inside journals, human readable text files configuring input/output folders, the sequence of LOBSTER functions to apply to each input image and their associated parameters. Journals are processed by typing JENI in the console and selecting a journal from the file browser:

  • >> JENI;

For instance, select the journal 'Tissue_SegWaterTiles.jl' from /Journals/jl. This journal identifies cells from cell-to-cell junction label. When launching a journal, every input image is displayed together with the results. Simply press 'x' to proceed to the next image. You can tweak the parameters of a journal: this is illustrated in this video. There are 3 kinds of journals: 2D images (.jl), 3D images (.jls) and 2D/3D time-lapses (.jm).

LOBSTER Panel

To start LOBSTER Panel, type >> LOBSTER in the console. The panel enables to load up to 2 journals, trigger object measurements and export object models. The important items from the upper section are (from left to right):

  • Input images dimension (2D/3D)
  • Whether object models should be exported or not
  • ZRatio (ratio between slice spacing and pixel size, default to 1)

A context help is provided when hovering an item from the panel. The buttons on the right side are used to save/load a project and to exit. To load a journal, press the button Journal1 and select its file. For instance, open the 2D journal 'Tissue_SegWaterTiles.jl' from previous section. Input and output folders configured in the journal are parsed and displayed in fields I and O. These fields can also be manually edited if required.

To launch a journal press on the associated Run button. By default the journal is processed but nothing is shown. To visually check the input/output of the journal, toggle the button Batch located in the lower part of the Panel to Adjust. If the results are not satisfactory, you can interrupt the journal by pressing 'q', edit the parameters in the journal by clicking the associated Edit button, save the file (floppy disk or Ctrl+S) from MATLAB script editor, and relaunch the journal. Once the results are satisfactory, toggle back to batch mode and launch the journal to process all images from the input folder. Open a journal input and output folders by pressing Show In and Show Out.

To perform measurements on the objects identified by Journal1, set the upper drop-down selector (object mask folder) to O1 and the lower drop-down selector to 'Objs' (since here we measure cells, that is plain objects). Press Run Measure 1: a folder with .csv files matching the names of the input images should pop up. You can open these files in an external software or browse the first reports by clicking Show some Reports.

Next, open the project 'FISH.mat' (Load button). This project loads two journals: the first one identifies nuclei and the other identifies FISH markers (in two different image channels). The aim is to count the number of FISH markers inside each nucleus. Since both journal points to the same input folder, the channel selection is performed inside the journals. Later on you can refer to this page to get more information about journals, but it is not strictly necessary at this point.

Measure 1 object mask is set to O1 (nuclei mask) and the drop-down selector right below O1 is set to O2, Journal2 output (FISH seed mask). This second drop-down configures the intensity measurement channel, here used to count the number of FISH markers per nucleus (number of non null pixels).

Journals and measurements can be triggered individually manually, or all at once automatically by pressing Run All. Of course, regular intensity measurements (e.g. from I1) can also be performed. If the images from the intensity channel hold the string '_C01' in their file names, the channel filter (field to the right of intensity measurement channel) should be set to '*_C01*.tif' instead of the default '*.tif'. This is for instance illustrated in the project 'NucleiIntensityMeas.mat'.

Sample Projects

Several sample projects are described below. All of these projects can be found in /Projects folder.

Tracing blood vessels from 3D images, exporting to SWC format and visualizing the model

Open project 'BloodVesselsTracing.mat', click Run All and visualize the output models (SWC filament networks) by loading the .SWC files from this webpage.


Segmenting nuclei from 3D images, exporting to STL format and visualizing the mesh

Open project 'CellPilar3DMesh.mat', click Run All and visualize the output models (STL mesh) by drag and droping them to this web page. For larger meshes, it is recommended to use MeshLab.


Detecting nuclei from 3D images, exporting the markers to CellInsight and editing them

Open project 'LymphGlandCellInsight.mat', click Run All. Open ImageJ and drag and drop the macro CellInsight (Tools/CellInsight/CellInsight.ijm). The first time, you need to copy the file GetString.ijm (located in the same folder) to ImageJ macros folder. Drag and drop the input image to ImageJ. Launch CellInsight.ijm and press Import to load the object markers. The object markers can be explored and edited (see CellInsight documentation). Large images can be opened in virtual stack mode (drag and drop image to ImageJ bar >> until Open as Virtual Stack is displayed).


Detecting nuclei from 3D images, exporting the markers as an ImageJ interactive montage

Drag and drop the report file (.csv) from the previous project to ImageJ and run the macro Tools/Montager.ijm. This builds an interactive montage of all detected objects: clicking an object jump to its position in the original image. The input image can also be opened in virtual stack mode to handle images of unlimited size.


Segmenting 2D objects from a time-lapse, tracking them and exploring the results in ImageJ

Open project 'HeLaTracking.mat'. This project performs tracking of dividing cells from a sequence of 2D images (TL is enabled to specify that Journal2 is a time-lapse journal). As can be noticed, I2 points to O1 since the 2D images are first segmented into object masks by the first journal before being tracked by the second journal. Measuring the output of the second journal brings 3 report files: area, CMx and CMy (object centroids X and Y position in the images). The results can be visualized by running the ImageJ macro Tools/OverlayTrackedObjects.ijm and selecting the input from the first journal as input folder and the output of the second journal as output folder (label mask).

You will notice a small error for the cell on top left in the first time point. This is because the tracker expects that the segmentation mask of first time point is manually corrected. You can do it before launching Journal2 by opening the first frame (click Journal1 Show Out) and use the macro Tools/MergeParticles to assist correction: Install the macro (Plugins >> Macro >> Install...). Then use ImageJ wand tool to select the pairs of particles to be merged and press F1. Save the image after edition and Run Journal2 and Measure 2 (do not Run All! or output from Journal 1 will be overwritten).


Processing large 3D images

To process 3D images of unlimited size, it is necessary to configure journals to do so by defining the variable Brick. Open project 'CellPilar_brcks.mat' for an example. In this project the sample image is not large but the project is configured to process large images. If you edit Journal 1 you will notice that the variable Brick is defined and set to 128 and that the variable GaurdBand is defined and set to 64. This instructs JENI and IRMA to process the image by bricks of size 128 x 128 x Number of slices of the original image, with an overlap of 64 pixels between bricks.

Using LOBSTER for your own Application

Designing practical journals and becoming familiar with LOBSTER functions can take some practice. To get started, the best approach is probably to test and adapt existing journals since many sample journals and companion images are provided (see LOBSTER documentation Appendix - Applications for the complete list). Here is how you can process:

  • Look for a journal performing at least part of the analysis you plan
  • Run the journal to see the input images and get a feeling about the results
  • Make a copy of the journal and modify it to point to your input image folder (for large images, you probably first want to crop out some illustrative regions)
  • Make sure that your images have the same bit depth than the images of the journal, if not optionally convert them.
  • Run the journal
  • You will probably need to tweak the parameters of some of the function used in the journal to obtain satisfying results. For this, open the journal and edit the parameters as was shown in previous sections. To get help on a function and its parameters, right click its name from MATLAB script editor and select Help on Selection
  • Completing the project might finally require some customization, such as combining multiple journals and performing cross-measurements or exporting object models (such as illustrated in the previous section)
