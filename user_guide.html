LOBSTER

Getting Started

Last updated: February 24th, 2020

LOBSTER is an image analysis environment to identify and optionally track biological objects in 2D/3D microscopy images and measure their spatial location, geometry, dynamics and intensity distribution. Multiple images can be batch processed in one go and the software sets no limit on image size. LOBSTER brings multiple viewers to explore and validate the results and the objects can be exported as 2D/3D models, for instance for their exploration or simulation in an external software.

Setting Up

Assuming that MATLAB (version >=2015a) and Image processing + Statistics & Machine Learning toolboxes installed:

  • Download LOBSTER from this GitHub repository (green button)
  • Unzip content to an empty folder where MATLAB has read/write access
  • Create an empty folder /Images inside installation folder and unzip the content of this archive inside (sample images)
  • Create an empty folder /Results inside installation folder, and unzip the content of this archive inside (sample results)
Windows is highly recommended, but LOBSTER should be compatible with all major operating systems. Refer to documentation for details (sections Requirements, Installation and Startup).

Initializing

LOBSTER should be initialized before using any of its components:

  • Launch MATLAB
  • >> cd E:/LOBSTER (replace with local LOBSTER installation folder to navigate to this folder)
  • >> init
The procedure is illustrated at the beginning of this video.

Architecture

Objects identification, measurements and exportation are performed sequentially and the results are stored in simple formats at every step. These operations are respectively performed by three computational modules described below, and called from MATLAB console (scripting) or LOBSTER Panel. In this tutorial, we cover the second option, refer to this page for scripting.

JENI: Identifying objects

TIFF images from an input folder are typically processed into object masks by applying a sequence of image analysis functions (workflow) configured in a journal (configuration file). The object masks can be of four kinds (see figure above):

  • Binary masks ('Objs'): objects encoded as connected groups of foreground pixels over 0 background
  • Skeleton masks ('Skls'): 1-pixel curves tracing filament centrelines
  • Seed masks ('Spts'): each foreground pixel (non zero) mark an object
  • Label masks ('Trks'): objects encoded as connected groups of pixels (unique ID) over 0 background.

IRMA: Measuring objects

Object masks can be used to measure objects or as the building blocks of more complex workflows. For instance, Object mask 1 , is used to measure C01 intensity and count the number of C02 objects inside C00 objects. Here Object mask 2 is a seed mask where each identified spot is encoded as a foreground pixel. Objects masks can also be used for object based co-localization study (mask overlap is computed), or to extract relative object distance distributions, e.g. spots to filaments (distance transform is computed from skeleton mask and read at identified spot locations). All object measurements are performed from object masks by the module IRMA, and results are exported to .csv report files.

JOSE/IRMA: Exporting object models

Identified objects can be exported as models: filaments networks are exported as SWC or OBJ models, object surfaces are exported as STL meshes and object tracks are exported as SWC trees or as Trackmate/Mamut/Mastodon tracks. Finally, object markers can be exported as CellInsight files (.xls), an ImageJ macro (provided in /Tools folder) enabling to visualize, classify and edit 3D point clouds.

Identifying Objects

The image analysis workflows used to identify objects are configured from journals, human readable text files configuring input/output folders, the sequence of LOBSTER functions to apply to each input image and their associated parameters. Journals can be call from LOBSTER Panel, or directly from MATLAB console by typing JENI:

  • >> JENI;

When doing so, a file browser pops up. Select for instance the journal 'Tissue_SegWaterTiles.jl' from /Journals/jl, which identifies cells from cell-to-cell junction label. When launching a journal, every input image is displayed together with the results. Simply press 'x' to proceed to the next image.

You can tweak the parameters of a journal by editing in a text editor (e.g. Matlab editor): this is illustrated in this video.

There are 3 kinds of journals with different extensions: 2D images (.jl), 3D images (.jls) and 2D/3D time-lapses (.jm).

LOBSTER Panel

LOBSTER Panel is probably the easiest way to get started, and almost all operations can be performed from this simple graphical user interface as long as no more than 2 journals are involved in a complete workflow.

To start the Panel, type >> LOBSTER in MATLAB console. Up to 2 journals can be loaded and object measurements / exportation can be performed from the outputs of the journals.

The most important items to configure first from the upper section of the Panel are (from left to right):

  • Input images dimension (2D/3D)
  • Whether object models should be exported or not
  • ZRatio, the ratio between slice spacing and pixel size (only relevant for 3D images).

For the remaining items, refer to the context help provided when hovering over them.

The buttons on the right side are used to save/load a project and to cleanly exit the Panel.

To load a journal, click the button Journal1 and select it from the file browser. For instance, load the 2D journal 'Tissue_SegWaterTiles.jl' used in the previous section. The input and output folders configured in the journal are automatically parsed and displayed in the fields I and O. These fields can also be set from a file browser by clicking I1/I2/O1/O2, or be manually edited. This is a convenient way to re-use journals to process different images.

To launch a journal, click the associated Run button. By default the journal is processed but nothing is shown (batch mode), this behaviour can be toggled from the button Batch located in the lower part of the Panel. If the results are not satisfactory, interrupt the journal by pressing 'q', edit its parameters by clicking associated Edit button, save the file (floppy disk or Ctrl+S) from MATLAB editor, and relaunch the journal. Once the results are satisfactory, toggle back to batch mode and launch the journal to process all images from the input folder.

The input and output folders of a journal can be browsed by clicking Show In and Show Out.

To perform measurements on the objects identified by Journal1, set the upper drop-down selector (object mask folder) to O1 (that is, the output of Journal 1) and set the lower drop-down selector to 'Objs' (plain objects measurements). Press Run Measure 1. Upon completion, press Show Reports to browse the folder holding the .csv reports.

It is possible to open an existing project by clicking Load button, for instance load the project 'FISH.mat'. This project loads two journals: the first journal identifies nuclei (DAPI channel) and the the second identifies FISH markers (as object seeds). The aim is to count the number of FISH markers inside every nucleus. Since both journals point to the same input folder, channel selection (DAPI/FISH) is performed inside the journals. To understand how, you can refer to this page to get more information about journals, but this is not strictly necessary at this point.

Measure 1 is set to O1 (nuclei mask) and the drop-down selector right below O1 (channel folder) is set to O2, that is the output of Journal2 (FISH seed mask). The bottom drop-down configures the intensity measurement channel, here used to count the number of FISH markers per nucleus (read from the report column Number of non null pixels).

This is a special case to illustrate the use of two journals but regular object intensity measurements (e.g. from I1) can also be performed. If the images from channel to be measured hold the string '_C01' in their file names, the channel filter (the field to the right of intensity measurement channel) should be set to '*_C01*.tif' instead of the default '*.tif'. This is for instance illustrated in the project 'NucleiIntensityMeas.mat'.

Finally, journal calls and measurements can be all triggered sequentially by pressing Run All. It is advised not to interact with the Panel while an operation is performed, this can be checked from the red color of the Run All button and from the log bar to the right.

Sample Projects

Several sample projects are described in this section, all can be found in LOBSTER /Projects folder.

Tracing blood vessels from 3D images

Tracing blood vessels from 3D images, exporting to SWC format and visualizing the 3D models.

Load project 'BloodVesselsTracing.mat', click Run All and visualize resulting .SWC files by loading them from this webpage.


Segmenting nuclei from 3D images

Segmenting nuclei from 3D images, exporting to STL format and visualizing the meshes.

Load project 'CellPilar3DMesh.mat', click Run All and visualize the STL meshes by drag and dropping them to this web page. For larger meshes, it is recommended to use MeshLab.


Detecting nuclei from 3D images

Detecting nuclei from 3D images, exporting the markers to LOBSTER CellInsight and editing them.

Load project 'LymphGlandCellInsight.mat', click Run All. Open ImageJ and drag and drop the macro CellInsight (click TOOLS button to access tools folder).

Note: The first time you use CellInsight, it is required to copy the file GetString.ijm to ImageJ /macros folder.

Drag and drop the input image to ImageJ (click Show In to browse input folder). Launch CellInsight.ijm from ImageJ macro editor, and press Import to load LOBSTER object markers. The object markers can be explored and edited (see CellInsight documentation). Exit CellInsight by clicking Exit, not by closing the image in ImageJ.

Note: Large images not fitting in main memory can be opened in virtual stack mode (drag and drop the image to ImageJ bar >> until Open as Virtual Stack is displayed).


Exporting detected nuclei as an ImageJ interactive montage

Drag and drop the report file (.csv) from the previous project to ImageJ, open the input image and run the macro Tools/Montager.ijm. This builds an interactive montage of all detected objects: clicking an object in the montage jumps to its position in the original image. The input image can also be opened in virtual stack mode to handle large images not fitting in main memory (drag and drop the image to ImageJ bar >> until Open as Virtual Stack is displayed).


Tracking dividing nuclei from a 2D time-lapse

Segmenting 2D objects from a time-lapse, tracking the objects, exploring results in ImageJ and plotting in Matlab.

Load project 'HeLaTracking.mat'. This project performs the segmentation and tracking of dividing nuclei from a sequence of 2D images. TL is enabled to specify that Journal2 is a time-lapse journal perfroming tracking (.jlm extension).

This time, I2 points to O1 since the images are first segmented into object masks (Journal 1) before being tracked (Journal 2).

When running the first journal, an image window pops up at the end of the processing. This is because the tracker (Journal 2) requires the best possible segmentation mask at first time point. To this end, the user needs to correct erroneous object splitting (most common errors) by drawing small rectangles around false cut lines. Beware to maximize the image window (or zoom around), to make sure that all objects are properly merged. Exit mask correction by double clicking on the image.

Measuring O2 (Object masks) brings 3 report files: objects areas, CMx and CMy (objects centroids). Tracking results can be visualized from ImageJ macro Tools/OverlayTrackedObjects.ijm by selecting O1 as input folder and O2 as output folder.

To plot some of the measurements, type PlotTrksMeasurements in MATLAB and select the .csv file of the measurement you want to plot. Type the ID of the objects you want to plot.


Tracking nuclei from a 3D time-lapse

Segmenting 3D objects from a time-lapse, tracking the objects and exploring the tracks in ImageJ Mastodon.

Load project 'Mastodon3D_Test.mat' and click Run All. To visualize the 3D tracks, an amazing tool is ImageJ Mastodon, leveraging the awesome ImageJ Big Data Viewer. To use Mastodon, it is first required to convert the input image to Big Data Viewer format by opening the image in ImageJ, making sure the dimensions are right from Image>Properties (Z and T can sometimes be swapped), and calling Plugins > BigDataViewer > Export Current Image as XML/HDF5. It is recommended to export the image to the same folder as the input image folder and with the default name (export.xml).

LOBSTER tracks can be converted to a format importable by Mastodon by typing MamutExport in Matlab console, and selecting the required files/folders: the XML input file should be in Images/Mastodon3D/export.xml and tracking / report folders are respectively O2 and R2 for this project.

Upon conversion, open Mastodon from ImageJ. Click New project in Mastodon window and select the input image XML file (export.xml). Click import mamut and select export_mamut.xml. Click bdv and trackscheme access to the image and track viewers. It is possible to edit the tracks and perform other operations. Refer to Mastodon documentation for reference.

Note: If you want to try Mastodon with object divisions, export the time-lapse from the previous project as XML/HDF5 and perform the same operations.

Processing large 3D images

To process large 3D images, bricking should be enabled by defining the variable Brick in the journal. For example, load the project 'CellPilar_brcks.mat' and inspect the journal (Edit): the variable Brick is set to 128 and the variable GaurdBand is set to 64. Both JENI and IRMA process the image by bricks of size 128 x 128, with an overlap of 64 pixels between bricks.

If needed, it is possible to convert bricked output masks to regular masks (folder of image slices) by typing ExportBricksAsSlices in MATLAB console. Select one of the folders (output image) from the output folder (here O1) and an empty folder to export the slices.

Using LOBSTER for your own Application

Designing practical journals and becoming familiar with LOBSTER functions can take some practice. To get started, the best approach is probably to test and adapt existing journals since many sample journals and companion images are provided (see LOBSTER documentation Appendix - Applications for the complete list). Here is how you can process:

  • Look for a journal performing at least part of the analysis you plan
  • Run the journal to see the input images and get a feeling about the results
  • Make a copy of the journal and modify it to point to your input image folder (for large images, you probably first want to crop out some illustrative regions)
  • Make sure that your images have the same bit depth than the images of the journal, if not optionally convert them.
  • Run the journal
  • You will probably need to tweak the parameters of some of the function used in the journal to obtain satisfying results. For this, open the journal and edit the parameters as was shown in previous sections. To get help on a function and its parameters, right click its name from MATLAB script editor and select Help on Selection
  • Completing the project might finally require some customization, such as combining multiple journals and performing cross-measurements or exporting object models (such as illustrated in the previous section)
